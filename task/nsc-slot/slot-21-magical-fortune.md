# Slot 21 (Magical Fortune) 개발

**진행 기간**: 2024.06 ~ 2024.12

---

## 배경

팀에 합류하고 처음 담당한 신규 슬롯이다. Magical Fortune은 텀블링(Cascading) 메커니즘과 와일드 스프레드가 조합된 슬롯이다.

기존 슬롯들과 다른 점이 많아서 처음에 도메인 파악하는 데 시간이 걸렸다.

---

## 핵심 메커니즘

### 텀블링 (Tumbling / Cascading)

당첨 심볼이 터지고 빈 자리를 위에서 새 심볼이 채우는 방식이다. 새 심볼이 채워진 뒤 다시 당첨 조건을 검사한다. 연속 당첨이 가능하다.

```
초기 릴:   [A][B][A]
           [C][A][A]  ← A 3개 당첨
           [D][E][F]

A 제거:    [ ][ ][ ]
           [C][ ][ ]
           [D][E][F]

심볼 낙하:  [?][?][?]  ← 새 심볼 채워짐
           [C][?][?]
           [D][E][F]
```

텀블 결과를 클라이언트에 내려줄 때 캐스캐이딩 횟수를 포함해야 했다. 처음에는 캐스캐이딩된 심볼 카운트를 내려줬는데, 스펙 변경으로 횟수로 바꿨다.

### 와일드 스프레드 (Wild Spread)

와일드 심볼이 나올 때 인접 셀에 추가로 와일드가 퍼지는 기능이다. 스프레드 범위가 설정에 따라 달라진다.

스펙이 개발 중반에 바뀌었다. 처음 구현 후 와일드 스프레드 로직을 스펙에 맞게 재작성했다.

### 보너스 랜덤 트리거

특정 조건에서 보너스 피처가 랜덤하게 발동된다. 랜덤 트리거가 발동되면 기존 윈도우를 그대로 유지하면서 추가 처리를 해야 했다.

처음에는 기존 윈도우를 응답에 포함하지 않았는데, 클라이언트에서 필요하다고 해서 추가했다.

---

## 구현하면서 있었던 일들

**aliasTable 위치 이동**

처음에 `aliasTable`을 `slotExtra`에 넣었는데 구조상 맞지 않았다. `reelExtra`로 이동했다. 이미 배포된 데이터가 있어서 마이그레이션이 필요했다.

**OverMaxPay 응답 누락**

최대 당첨 금액 제한(`OverMaxPay`)이 걸렸을 때 클라이언트에 응답을 내려줘야 하는데 빠져 있었다. 응답 필드를 추가했다.

**SymbolPositions 동작 오류**

심볼 위치 정보를 담는 `SymbolPositions` 클래스의 동작이 예상과 달랐다. 텀블 이후 심볼 위치를 계산할 때 누락되는 케이스가 있었다. 로직을 수정하고 시뮬레이터에서 검증했다.

**시뮬레이터 init-spin, end-spin 미포함**

시뮬레이터가 일반 스핀 결과만 집계하고 있었다. init-spin(첫 스핀)과 end-spin(마지막 스핀) 결과를 집계에 포함하지 않으면 RTP가 실제와 달라진다. 항목을 추가했다.

**RTP 윈 레인지 집계 오류**

시뮬레이터에서 프리스핀 결과가 RTP 윈 레인지에 합산되지 않고 있었다. 프리스핀은 별도 루프로 돌아가는데 집계 로직에서 누락됐다.

**치트 미스 처리 오류**

치트 사용 시 미스된 릴 결과를 내려줄 때 가끔 미스가 아닌 결과가 섞여서 나왔다. tumbleReelResult를 반환에 포함하지 않아서 생긴 문제였다.
