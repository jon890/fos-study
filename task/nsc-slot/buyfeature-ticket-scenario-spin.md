# BuyFeature 티켓 & 시나리오 스핀 구현

**진행 기간**: 2024.12

---

## 배경

두 가지 기능을 같은 시기에 개발했다. 둘 다 슬롯 플랫폼 공통 기능이다.

---

## BuyFeature 티켓

### 기능 개요

유저가 재화를 내고 피처(보너스 라운드)를 즉시 구매하는 기존 BuyFeature에 더해, **티켓**으로 바이피처를 사용하는 기능이다.

이벤트 등으로 획득한 티켓을 사용하면 재화 없이 피처에 진입할 수 있다.

### 구현 포인트

**티켓 사용 후 베팅 인덱스 복귀**

바이피처 티켓은 클라이언트가 요청한 베팅 인덱스를 기준으로 피처에 진입한다. 피처가 끝나고 베이스로 돌아올 때 어떤 베팅 인덱스로 복귀할지가 처음에 불명확했다.

클라이언트가 요청한 베팅 인덱스를 피처 진행 내내 유지하고, 베이스 복귀 시에도 동일 인덱스를 반환하도록 했다.

**튜토리얼 대응**

튜토리얼 흐름에서도 티켓 API를 사용하는 케이스가 있었다. API 변경에 따른 튜토리얼 코드 대응이 필요했다.

**바이피처 옵션 없는 경우 에러코드**

바이피처를 지원하지 않는 슬롯에서 바이피처를 요청하면 명확한 에러코드를 반환하도록 추가했다.

## 시나리오 스핀

### 기능 개요

미리 정의된 스핀 결과를 순서대로 재생하는 기능이다. 튜토리얼이나 데모 목적으로 쓴다.

```
시나리오 정의:
  1번째 스핀: [A][A][A] → 소액 당첨
  2번째 스핀: [B][B][B] → 보너스 진입
  3번째 스핀: (보너스 스핀 결과)
  ...

시나리오 스핀 실행:
  스핀 요청 → 정의된 결과 순서대로 반환
```

### 구현 포인트

**빅윈 시나리오**

튜토리얼에서 빅윈이 발생하는 시나리오를 시연하기 위한 기능이다. 빅윈 스핀 결과를 시나리오로 만들어 재생했다.

**슬롯별 시나리오 통합**

각 슬롯이 시나리오를 따로 처리하고 있었다. 시나리오 스핀을 플랫폼 레벨에서 통합해서 슬롯별로 직접 구현하지 않아도 되도록 했다.

통합 과정에서 도지(Doge), 그래피티(Graffiti) 슬롯 전용 시나리오도 추가했다.

**튜토리얼과의 통합**

기존 튜토리얼에서 시나리오를 불러오는 방식을 통합된 시나리오 스핀 API를 쓰도록 변경했다. 도지 시나리오 수행 시 buyFeature → free 변환 처리도 수정했다.

**튜토리얼 첫 요청 검증**

튜토리얼 첫 요청 시 베이스 스테이지가 아니면 에러를 던지도록 추가했다. 진행 중인 게임에서 잘못 튜토리얼이 시작되는 케이스를 방지하기 위해서다.
