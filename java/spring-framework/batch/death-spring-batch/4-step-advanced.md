# 4장 - 스텝 해부 작전

이제 배치 스텝의 종지부를 찍어보자.

### 먼저 숨겨진 비밀, ItemStream의 정체를 밝혀보자.

리소스 관리부터 실행 관리까지, Spring Batch의 신뢰성을 지탱하는 이 핵심 인터페이스가 어떤 역할을 하는지,
그리고 이를 다룰 때 발생할 수 있는 함정까지 알아볼 것이다.

### 그 다음은... ItemProcessor

단순한 가공 로직처리를 할 수 도 있지만, 잘못 다루면 치명적인 성능 이슈가 발생할 수 있다.
제대로 다루는 법을 알아보자.

### 마지막은 FaultTolerance...

실전에서 문제가 터졌을 떄 당황하지 않을 수 있도록, 내결함성의 핵심을 정확하고 철저하게 파헤쳐보자.

### 최종으로 스텝 내부를 파헤쳐보자.

내부 구현은 실무에서 배치 애플리케이션을 충분히 경험해본 후에 살펴보는 것이 더 효과적일 것이다.
하지만 시스템을 진정으로 이해하고 싶다면, 한 번 정도는 스텝의 핵심을 직접 들여다볼 가치가 분명히 있다.
